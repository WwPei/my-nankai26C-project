## 一、项目概览

**Emoji Dungeon** 是一款基于 **Qt6 Widgets** 框架开发的 **2D 俯视角地牢战斗游戏**（阶段0/1骨架）。项目使用 `QGraphicsScene` / `QGraphicsView` 构建战斗场景，采用纯 Widget 体系（非 QML），整体遵循 **MVVM（Model-View-ViewModel）** 类似的架构思想，将数据逻辑与视图渲染分离。

## 二、代码框架与架构设计

### 2.1 整体架构图

```
main.cpp (入口)
    └── EmojiDungeonWindow (主窗口 / 页面导航器)
            ├── QStackedWidget (页面栈)
            │   ├── StartPage      (开始页 - 内联创建)
            │   ├── ClassSelectPage (职业选择页)
            │   ├── GameMainPage   (战斗主页)
            │   └── UpgradePage    (升级选择页)
            └── GameFactory (实体工厂 - 全局注入)
```

### 2.2 核心设计模式

| 模式 | 应用位置 | 说明 |
|------|---------|------|
| **工厂模式** | `GameFactory` | 统一创建 Player/Weapon/Enemy/Bullet/Trait 实体 |
| **抽象接口** | 所有 `*.h` 头文件 | Data 层（逻辑）和 View 层（渲染）均定义纯虚接口 |
| **MVVM-like** | Data ↔ View 分离 | 如 `EnemyData`(逻辑) ↔ `EnemyView`(渲染) 通过信号槽绑定 |
| **状态机** | `PageId` / `BattleFlowState` | 管理页面切换和战斗流程状态 |
| **信号槽驱动** | 全局 | Qt 核心机制，用于实体间解耦通信 |

### 2.3 页面导航状态机

```
[Start] ──"开始游戏"──▶ [ClassSelect] ──"选择职业"──▶ [GameMain]
   ▲                         │                           │
   │◀──"返回"────────────────┘                           │
   │                                              "打开升级页"
   │                                                     ▼
   ◀────────────"返回开始页"──────────────────── [Upgrade]
                         ◀──────"返回战斗"──────────────┘
```

## 五、关键技术实现要点

### 5.1 Data-View 分离绑定

所有游戏实体严格遵守"数据驱动视图"模式，以 Enemy 为例：
```
EnemyData (纯逻辑)
   ├── 属性：位置/生命值/移速/碰撞半径
   ├── 方法：advanceFrame()/receiveDamage()
   └── 信号：positionChanged / defeated
         │ (connect)
   EnemyView (纯渲染)
      ├── bindModel(EnemyData*) → 建立绑定
      ├── syncFromData() → 同步位置+外观
      └── paint() → QPainter 绘制
```

### 5.2 武器开火系统

- `Weapon::advanceCooldown(deltaMs)` 通过累积倒计时实现射击节奏
- 支持多弹丸散射（SpreadBlaster 的 `projectileCount=3`），通过 `rotateVector()` 按12°步进展开弹道
- 按住左键持续射击（`startFiring()`/`stopFiring()`），松开即停

### 5.3 碰撞检测

- 子弹-敌人碰撞：通过 `QGraphicsItem::collidingItems()` 检测重叠，子弹命中后立即 `expire()`
- 敌人-玩家碰撞：通过 `enemy.view->collidesWithItem(m_playerMarker)` 检测，玩家有 600ms 受伤冷却

### 5.4 暗色主题

战斗页面使用内联 QSS 样式表（`kBattlePageStyle`），定义了完整的暗色配色方案（背景 `#1b1d21`，面板 `#20242b`，强调色 `#4b7bec`），进度条和按钮均有自定义 hover/pressed 状态。

### 5.5 测试直接访问私有成员

测试文件通过 `#define private public` 技巧临时暴露 `EmojiDungeonWindow` 和 `GameMainPage` 的私有成员以进行白盒测试，这种模式适用于快速原型阶段。
