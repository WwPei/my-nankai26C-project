## 一、总体变化概览

V2 相对于 V1 是一次**从"草稿原型"到"可扩展架构"的全面重构**。核心变化在于：

| 维度 | V1 | V2 |
|------|----|----|
| **架构理念** | 快速原型：逻辑与渲染混合 | 严谨分层：Data/View/Factory 三层分离 |
| **代码组织** | 每个实体独立 .h/.cpp | 抽象接口(.h) + 具体实现集中在 game_factory.cpp |
| **数据类型** | 简单 struct（ClassData） | 枚举类 + 7种Config结构体 + 静态数据表 |
| **扩展能力** | 硬编码逻辑，添加新实体需多处修改 | 抽象接口+工厂，新增实体只需加配置即可 |
| **测试覆盖** | 无 | 5个系统级测试用例（QtTest） |
| **视觉呈现** | 基础矩形+纯色 | 自定义QPainter绘制 + 暗色QSS主题 |
| **CMake构建** | Qt5/Qt6双兼容 | 明确要求 Qt6.10+，双目标构建 |

#### game_factory.h/.cpp

**V1 工厂**：~850 行代码分布在 game_factory.cpp + 9个独立实体 .cpp 文件中
- 构造函数需要 `QGraphicsScene*` 依赖
- `createEnemyData(int typeId)` - 只接受 int 类型 ID
- `createBulletData(spawnPos, dir, baseDamage, weapon)` - 伤害计算在工厂内部
- 工厂自动将 View 添加到场景（`m_scene->addItem(view)`）

**V2 工厂**：~860 行全部集中在 game_factory.cpp 的匿名命名空间
- 构造函数无参数（零依赖）
- 返回组合实体 `EnemyEntity {data, view}` / `BulletEntity {data, view}`
- 全部使用枚举 ID（`PlayerClassId`/`WeaponId`/`EnemyId`/`TraitId`）
- 工厂**只创建不装配**（不操作场景），完全解耦
- 新增 `EnemyEntity`/`BulletEntity` 结构体便于管理
- 新增 `createPlayer()`/`createStarterWeapon()`/`createTrait()` 等全套方法
- 明确定义了所有具体实现类（匿名命名空间内的 BasicPlayer/BasicWeapon/...等7个类）

#### 实体类层次结构

```
V1 类层次（混合式）:
  QObject ─── Weapon      (独立数据类)
  QGraphicsRectItem ─── PlayerItem  (数据+视图混合)
  QGraphicsRectItem ─── EnemyView   (视图，直接持有 data)
  QObject ─── EnemyData   (独立数据)
  QGraphicsEllipseItem ─── BulletView  (视图)
  QObject ─── BulletData  (独立数据)
  QObject ─── Trait       (特性，接口式但不是纯虚)

V2 类层次（完全分离）:
  数据层 (QObject):
    Player        ← BasicPlayer      (game_factory.cpp 内)
    Weapon        ← BasicWeapon
    EnemyData     ← BasicEnemyData
    BulletData    ← BasicBulletData
    Trait         ← BasicTrait

  视图层 (QGraphicsObject):
    GameView        ← EnemyView    ← BasicEnemyView
                 ← BulletView   ← BasicBulletView
```

## 三、架构演进总结

### 3.1 V1 架构问题（V2 已解决）

1. **数据视图耦合**：`PlayerItem`/`EnemyView` 同时持有数据和渲染，拓展困难
2. **硬编码配置**：所有数值散布在代码中（`if (typeId == 1) hp=50`）
3. **工厂依赖场景**：`GameFactory` 需要 `QGraphicsScene*`，创建即添加，不够灵活
4. **无测试**：缺少自动化验证
5. **单一构建目标**：没有测试可执行文件
6. **命名不规范**：信号用 `sigXxx`（Qt4 风格），V2 统一为 Qt 现代风格
7. **无状态管理**：`m_isGamePaused`/`m_isGameActive`/`m_waveEnding` 多个 bool，V2 统一为 `BattleFlowState` 枚举

### 3.2 V2 架构优势

1. **纯虚接口体系**：所有实体定义标准接口，具体实现可互换
2. **配置驱动**：`game_data.h` 单文件管理全部游戏数值，调整无需改逻辑
3. **工厂解耦**：工厂只创建实体，不管理场景生命周期
4. **暗色主题**：完整 QSS 定义，战斗页视觉表现大幅提升
5. **自动化测试**：5个核心测试覆盖战斗流程
6. **碰撞系统升级**：利用 Qt 内置 `collidingItems()` 代替手动距离计算
7. **时间驱动**：全部移动/冷却使用 `deltaTime`，帧率无关
8. **雷达图式扩展**：新增职业/武器/敌人只需在 `game_data.h` 加一行配置

## 五、从 V1 到 V2 的关键决策

1. **废弃 Qt Designer UI**：移除 `mainwindow.ui`，纯代码构建界面，更可控
2. **实体 .cpp 集中化**：将 9 个独立实体 .cpp 文件合并到 `game_factory.cpp` 匿名命名空间，减少编译单元、降低文件管理复杂度
3. **引入枚举类替代 int ID**：类型安全、IDE 友好
4. **经验/等级系统预留**：数据结构完整定义但未接线，为 V3/V4 扩展做准备
5. **波次管理器接口**：`wave_manager.h` 定义了标准接口但未实现，V3 可接入
6. **删除音频预留代码**：精简 `class_select_page`，去掉 V1 中的注释式音频占位
7. **窗口尺寸变更**：从 1280×720 固定改为 1080×720 可调整
