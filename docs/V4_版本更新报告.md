# Emoji Dungeon V4 版本更新报告

> 发布日期：2026-05-11\
> 基于：emoji_dungeon_v3 项目 57 个源文件\
> 构建：CMake + Qt 6.10.2 MinGW / C++17

---

## 一、版本概述

V4 版本在 V3 奠定的 **7 类型 MV + 子系统** 架构基础上，聚焦**完整可玩性**和**视听表现**两个维度进行了大规模重构。本版本主要完成：UI 界面全面重制、游戏数值深度优化、视听系统升级、以及多项稳定性修复。

### 2.4 升级弹窗替代页面跳转

**涉及文件：** [game_main_page.cpp](file:///F:/QT%20projects/emoji_dungeon_v3/src/pages/game_main_page.cpp)

| 维度 | V3 | V4 |
|------|----|----|
| 呈现方式 | QStackedWidget 整页切换 | 半透明悬浮弹窗覆盖战斗画面 |
| 流程 | 3 层信号转发 | 直接内联 lambda 处理 |
| 动画 | 无 | 淡入 350ms OutCubic / 淡出 280ms InCubic |

## 三、页面导航重构

### 3.1 删除 UpgradePage 独立页面

**涉及文件：** [emoji_dungeon_window.h](file:///F:/QT%20projects/emoji_dungeon_v3/src/pages/emoji_dungeon_window.h) / [emoji_dungeon_window.cpp](file:///F:/QT%20projects/emoji_dungeon_v3/src/pages/emoji_dungeon_window.cpp) / [game_enums.h](file:///F:/QT%20projects/emoji_dungeon_v3/src/core/game_enums.h)

- `showUpgradeOverlay()` 增加防重入检查 `isVisible()`
- 调用延迟一帧执行 `QTimer::singleShot(0, ...)` 防止战斗 tick 内的重入崩溃

## 六、Bug 修复汇总

| 问题 | 根因 | 修复 |
| ---- | ---- | ---- |
| 升级弹窗点击崩溃 | `qApp->installEventFilter(this)` 全局事件捕获冲突 + `QGraphicsOpacityEffect` 与 QGraphicsView 渲染管道冲突 | 删除 `qApp` 事件过滤器；`QVariantAnimation` + stylesheet alpha 替代 |
| QSoundEffect MP3 解码错误 | Qt 6.10 FFmpeg 后端不兼容 QSoundEffect 的 MP3 解码 | 彻底删除 QSoundEffect，统一使用 QMediaPlayer |
| 升级后 WASD 仍然移动 | `setBattleState(Upgrade)` 未设置 | 先 `setBattleState(Upgrade)` 再 `setBattleActive(false)` |
| Boss 血条每帧抖动 | 追踪 Boss 场景坐标有不稳定延迟 | 改为固定定位到波次进度条下方 |
| 角色/子弹拖尾残影 | `MinimalViewportUpdate` 仅重绘脏矩形 | `FullViewportUpdate` 单行修复 |
| 背景图片不显示 | `paintEvent` 被子控件覆盖 | 移至 `BattleArenaView::drawBackground()` |
| 机器人不射击 | `shootTriggerDistance` 被误设为 `true` | 改为 `0.0F` |
